杂

  1. Jump end 动作没做完，就按下空格，导致使用连跳时，会先播放ground动作，天上跑

    • 从 end 增加 jumping 触发条件到 loop 动作，fall优先判断是否起跳，完美解决

  2. 物理引擎不计算图层 edit -> project setting -> physics

  3. 按住shift奔跑改为长按方向

    • 在按住方向键时，开启计时器，如果达到规定时间，则isRun
    • isRun开启下，不计时
    • 松手一秒钟后isRun关闭

  4. 在编辑器配置好脚本所需列表参数，运行后，列表数量归0

    • 脚本中，列表List的初始化放在了start中导致，直接在声明时初始化解决了问题

  5. 在激活对象池对象时，需要重置位置，新对象却不调用重置方法

    • 调用方法所需变量目标点变换组件的赋值放在了Start里面，导致第二次激活时没有调用
      • Start只在开始时执行一次，对象禁用后，不会重新计算这个生命周期！！！
    • 重置方法放错位置，导致在新对象创建时，不会调用

  6. 带有碰撞体和刚体不触发碰撞

    • 设置层级关系

  7. 根据击中物体，生成特效

    • 对象池生成对象的名字：自定义命名规则，Effects + 身体部位
    • 再根据预制件名字通过资源管理类寻找

  8. OnCollisionXXX 子类可直接用父类，OnTriggerXXX子类需要重写才能用

  9. 值类型不会为空，除非加 ? 做可空值类型

  10. hit.normal 特效z轴与法线一致

  11. 特效重合导致若隐若现

    • 朝法线方向移动一点点

  12. AI在到达攻击范围后任然会保持追逐状态，导致原地踏步

    • 攻击目标消失条件判定的距离和到达攻击范围内判定的距离一个是大于，一个小于等于，导致可能出现无限在这两个状态间切换
    • 增加了一点点攻击目标消失条件判定的距离，使这两个不共用一个判定距离

  13. UI面板push多执行了OnStart方法，导致按钮时间多绑定了一次，点一次，执行了两次Pop

    • 调整调用OnStart方法的位置

  14. 场景切换，面板不出现

    • 面板生成在了第一个场景Canvas下，随着第一个场景的Canvas一起销毁了
    • 不销毁第一个Canvas

  15. Setting the parent of a transform which resides in a Prefab Asset is disabled to prevent data corruption

    设置预制体的父物体是不允许的

    • 在加载预制体后，未对其进行实例化

    • 原代码

    • 更改为

  16. 加上canvas后Button无法检测

    • https://blog.csdn.net/weixin_43673589/article/details/121236786

  17. Toggle组要实现循环设置监听，就要给各个Toggle设置相对应的编号，而不能用for循环里面的变量

    • ```c#

      {

  18. AB包打包不允许有任何报错，否则就打不出来

  19. 不能用File.Exits() 判断streamingAssets下的资源是否存在，永远返回false，网上的解释是因为streamingAssets本身是一个虚拟路径，并不真实存在

  20. 消息传递：

    • 广播消息 BroadcastMessage() ，广播给所有的孩子
      • 可降低代码的耦合性

    • sendMessageUpwards 向上发送

SQLite

准备

因为不同版本unity可能不兼容

Plugins文件夹下放入，mono.Data.Sqlite.dll（编辑器找）和libsqlite3和sqlite3.dll（官网下载）

操作

  • 每次操作都需要开启和关闭

  • 同名表单只能有一份

    • 连续创建即报错

  • 文本输入不是纯数字，必须加上单引号

  • 查询多条数据时，需要把读取代码放入循环中，直至读完

  • 直接在编辑器中删除数据库，是没法真正删除的

    • 关闭unity，在文件路径中删除

  • 发布版本同sqlite3同样64位

优化

降低Draw Calls数

动态批处理

  1. 网格顶点数300
  2. 材质相同且材质着色器必须是单pass
  3. 纹理不影响
  4. 必须是Mesh Renderer 渲染的组件
  5. 每个物体的阴影占用一个
  6. 所有静态物体不参与
  7. 缩放倍数相差大时无法参与

静态批处理

场景，建筑

  1. 必须是静态
  2. 和顶点无关
  3. pass无关
  4. 和缩放有关，过大则不参与

特殊文件夹

  1. standard Assets 此文件夹下脚本最先被编译
  2. Editor 允许里面的脚本访问 unity edior的API
  3. Plugins 插件目录，只能是Assets文件夹的直接子目录，优先编译
  4. Resources 里面的的文件不管有没使用，都会跟随游戏一起打包。可以使用Resources.UnloadUnusedAssets释放内存
  5. Gizmos，存放用Gizmos.DrawIcon方法使用的贴图，图标资源。放在Gizmos文件夹中的贴图资源可以直接通过名称使用，可以被Editor作为gizmo画在屏幕上

StreamingAssets：

  • 打包资源，二进制文件放在此文件夹下才能够随工程打包
  • 不压缩不加密
  • PC端是可读可写的

在移动端

  1. 不能使用FILE等等文件操作来访问，安卓沙盒采用协议地址，文件类找不到
  2. 要修改只能复制到PersistentDataPath下
  3. 只读

  • 几个平台下的路径：

PersistentDataPath：

  1. 可读可写

  2. 只能在运行时进行写入读取

移动

  1. 在安卓可以是程序的沙盒，也可以是sdcard

文件

PlayerPrefs 本地持久化

  • 需要创建一个【Key键名】然后通过PlayerPrefs.GetInt(Key)来获取【value值】
  • 每一个键名只能对应一个值，也就是说，保存变量有五个，就需要创建五个键
  • 不适用与存储相对复杂的内容
  • 适用于存储用户选项设置，图片，简单数值

序列化反序列化中二进制存储

  • 保存的文件因为是二进制，阅读困难

JSON

  • 轻量级的数据交换格式
  • 仅支持简单类型，不支持集合，如字典（但是支持List）
  • 体积小，速度快

XML

  • 可以将大量的信息通过XML元素的心事储存在xmlDocument中
  • 解析得考虑，根元素，子元素，加载过程还要通过 标签名找到xml元素的集合
  • 冗余信息较多，占用空间最大，文件格式较为复杂，解析文件需要花费较多资源和时间
