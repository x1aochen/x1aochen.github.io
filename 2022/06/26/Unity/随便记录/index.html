杂

  1. Jump end 动作没做完，就按下空格，导致使用连跳时，会先播放ground动作，天上跑

    • 从 end 增加 jumping 触发条件到 loop 动作，fall优先判断是否起跳，完美解决

  2. 物理引擎不计算图层 edit -> project setting -> physics

  3. 按住shift奔跑改为长按方向

    • 在按住方向键时，开启计时器，如果达到规定时间，则isRun
    • isRun开启下，不计时
    • 松手一秒钟后isRun关闭

  4. 在编辑器配置好脚本所需列表参数，运行后，列表数量归0

    • 脚本中，列表List的初始化放在了start中导致，直接在声明时初始化解决了问题

  5. 在激活对象池对象时，需要重置位置，新对象却不调用重置方法

    • 调用方法所需变量目标点变换组件的赋值放在了Start里面，导致第二次激活时没有调用
      • Start只在开始时执行一次，对象禁用后，不会重新计算这个生命周期！！！
    • 重置方法放错位置，导致在新对象创建时，不会调用

  6. 带有碰撞体和刚体不触发碰撞

    • 设置层级关系

  7. 根据击中物体，生成特效

    • 对象池生成对象的名字：自定义命名规则，Effects + 身体部位
    • 再根据预制件名字通过资源管理类寻找

  8. 值类型不会为空，除非加 ? 做可空值类型

  9. hit.normal 特效z轴与法线一致

  10. 特效重合导致若隐若现

    • 朝法线方向移动一点点

  11. AI在到达攻击范围后任然会保持追逐状态，导致原地踏步

    • 攻击目标消失条件判定的距离和到达攻击范围内判定的距离一个是大于，一个小于等于，导致可能出现无限在这两个状态间切换
    • 增加了一点点攻击目标消失条件判定的距离，使这两个不共用一个判定距离

  12. UI面板push多执行了OnStart方法，导致按钮时间多绑定了一次，点一次，执行了两次Pop

PlayerPrefs 本地持久化

  • 需要创建一个【Key键名】然后通过PlayerPrefs.GetInt(Key)来获取【value值】
  • 每一个键名只能对应一个值，也就是说，保存变量有五个，就需要创建五个键
  • 不适用与存储相对复杂的内容
  • 适用于存储用户选项设置，图片，简单数值

序列化反序列化中二进制存储

  • 保存的文件因为是二进制，阅读困难

JSON

  • 轻量级的数据交换格式
  • 仅支持简单类型，不支持集合，如字典（但是支持List）
  • 体积小，速度快

XML

  • 可以将大量的信息通过XML元素的心事储存在xmlDocument中
  • 解析得考虑，根元素，子元素，加载过程还要通过 标签名找到xml元素的集合
  • 冗余信息较多，占用空间最大，文件格式较为复杂，解析文件需要花费较多资源和时间
