const

  1. 要在其他文件使用变量，加extern
  2. const必须初始化，类中不能初始化，类中只能通过构造函数
  3. 对于非内部数据类型的输入参数，应该将“值传递”的方式改为“const 引用传递”，目的是提高效率。例如将void func(A a) 改为void func(const A &a)
  4. 对于内部数据类型的输入参数，不要将“值传递”的方式改为“const 引用传递”。否则既达不到提高效率的目的，又降低了函数的可理解性。例如void func(int x) 不应该改为void func(const int &x)
  5. const对象只能访问const成员函数
  6. 有些地方不加const编译不过

static

  • 在类对象实例化之前就已分配了空间，并且只分配一次
  1. static静态成员变量不能在类的内部初始化。在类的内部只是声明，定义必须在类定义体的外部，通常在类的实现文件中初始化
  2. 类中的静态变量由对象共享。对于不同的对象，不能有相同静态变量的多个副本
  3. 静态对象的范围是贯穿程序的生命周期
  4. 允许静态成员函数仅访问静态数据成员或其他静态成员函数，它们无法访问类的非静态数据成员或成员函数

inline

  1. 内联能提高函数效率，但并不是所有的函数都定义成内联函数！内联是以代码膨胀(复制)为代价，仅仅省去了函数调用的开销，从而提高函数的执行效率
  2. 如果函数体内代码较长（增加内存消耗）或者出现循环（效率减少），那么不宜用内联

sizeof

  1. 静态成员不影响类大小。
  1. 类的数据成员按其声明顺序加入内存（字节对齐）
  2. 返回值类型为size_t，
  3. size_t类型是一个类型定义，通常将一些无符号的整形定义为size_t，比如说unsigned int或者unsigned long，甚至unsigned long long

pure virtual

  1. 抽象类至少包含一个纯虚函数
  2. 不能创建抽象类对象，可以通过抽象类的指针或引用访问派生类对象
  3. 如果我们不在派生类中覆盖纯虚函数，那么派生类也会变成抽象类

vptr_vtable

  1. 每个使用虚函数的类（或者从使用虚函数的类派生）都有自己的虚拟表。该表只是编译器在编译时设置的静态数组。虚拟表包含可由类的对象调用的每个虚函数的一个条目。此表中的每个条目只是一个函数指针，指向该类可访问的派生函数

  2. vptr在创建类实例时自动设置，以便指向该类的虚拟表

  3. ```c++
    Base *p = new Drived;
    p->func();

  4. static成员不属于任何类对象或类实例，所以即使给此函数加上virutal也是没有任何意义的

  5. 静态成员函数没有this指针。所以无法访问vptr，进而不能访问虚函数表

  6. 构造函数不能是虚函数，而析构函数必须是虚析构函数（父类指针指向子类对象，因为是虚函数，编译器会忽略指针的类型，而根据指针的指向来选择函数，先调用子类，在调用父类）

  7. 唯一可以内联的时候是：编译器知道所调用的对象是哪个类 Base::func()

assert

  1. ASSERT ()是一个调试程序时经常使用的宏，在程序运行时它计算括号内的表达式，如果表达式为FALSE (0), 程序将报告错误，并终止执行。如果表达式不为0，则继续执行后面的语句。这个宏通常原来判断程序中是否出现了明显非法的数据，如果出现了终止程序以免导致严重后果，同时也便于查找错误

extern “C”

friend

  1. 友元函数：普通函数可以访问某个类中的私有或保护成员
  2. 友元类：类A声明friend类B，类B可以访问类A的私有或保护成员（单向）
  3. 友元关系不可传递
  • 优点：提高了程序的运行效率
  • 缺点：破坏了类的封装性和数据的透明性

define这是简单的字符串替换

  1. 字符串化操作符（#）
  • 在一个宏中的参数前面使用一个#,预处理器会把这个参数转换为一个字符数组，换言之就是：**#是“字符串化”的意思，出现在宏定义中的#是把跟在后面的参数转换成一个字符串**
  1. 符号连接符（##）
  • “##”是一种分隔连接方式，它的作用是先分隔，然后进行强制连接。将宏定义的多个形参转换成一个实际参数
  • 连接后的实际参数名，必须为实际存在的参数名或是编译器已知的宏定义
  • 如果##后的参数本身也是一个宏的话，##会阻止这个宏的展开
  1. 续行操作符（\）
  • 当定义的宏不能用一行表达完整时，可以用”\”表示下一行继续此宏的定义

  2. 用于检查自己重写的函数，是否条件正确

final

  1. 第一个用在类，用于说明该类是继承体系下最后的一个类，不要其他类继承我，当继承时就会报错
  2. 第二个用在虚函数，表示这个虚函数不能再被override了，再override就会报错
